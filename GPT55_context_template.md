# GPT-5.5 壁打ち用コンテキスト

このファイルは、HashimotoAddons の新機能について GPT-5.5 と壁打ちするときに渡す前提情報を整理するためのテンプレートです。

## 目的

- 機能案の整理
- Safe / Full の分類判断
- 実装方針の相談
- 影響範囲と検証観点の洗い出し

## プロジェクト前提

- 対象: Minecraft Java Edition 1.16.5
- 実装基盤: Fabric
- 開発対象は `Safe` と `Full` の2系統
- 変更は既存の設計と変数に合わせて最小範囲に抑える
- 互換性や永続化の変更は慎重に扱う

## 重要ルール

- 新機能は既定で無効にする
- サーバー敏感な挙動や自動化は原則 `Full`
- `Safe` は危険な UI、コマンド、キー割り当て、設定、動作を露出しない
- 分類が曖昧なら owner に確認する
- 実装前に `Safe` / `Full` / `Both` を明示する

## 既存の知識

- 既知の実装知見は `docs/KNOWLEDGE.md`
- 重要な設計判断は `docs/decisions/`
- 構造上の問題は `docs/TECHNICAL_DEBT.md`
- Safe/Full の永続ルールは `docs/SAFE_FULL_POLICY.md`

## 壁打ちで伝えるとよい情報

- 機能名
- 何を解決したいか
- 想定ユーザー
- 使う場面
- 既存機能との関係
- `Safe` / `Full` の希望
- 入力方法や UI が必要か
- 永続化が必要か
- サーバー状態の参照や自動操作があるか
- 失敗時の挙動
- 既存コードへの影響範囲
- 受け入れ条件

## GPT-5.5 に聞きたい観点

- この機能は `Safe` / `Full` / `Both` のどれにすべきか
- 依存を切り分けるならどこを pure logic にできるか
- 既存の設計に対して最小の変更は何か

2. 相談したい機能の情報を埋める
